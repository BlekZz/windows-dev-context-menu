# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Project Does

A Windows right-click context menu tool that adds a cascading "Dev Tools" submenu to Windows Explorer. Users right-click a folder background → Dev Tools → pick a terminal/editor/tool. The menu is registered per-user (`HKCU`) with no admin required.

## Script Execution

All scripts run with:
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\<script>.ps1
```

**Workflow order:**
1. `setup-env.ps1` — detect installed apps, write `.env`
2. `install.ps1` — register context menu in registry (auto-runs setup if `.env` missing)
3. `hide-duplicates.ps1` — suppress per-app duplicate entries (requires admin)
4. `uninstall.ps1` — remove menu and restore hidden entries (requires admin if restoring HKLM entries)

**Force re-detect paths:**
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup-env.ps1 -Force
```

## Architecture

### Data flow
```
setup-env.ps1 → .env (key=value pairs)
install.ps1   → reads .env → HKCU registry entries
                              each entry calls dev-launcher.ps1 -action <name> -path "%V"
dev-launcher.ps1 → re-reads .env → launches the app
```

### Registry location
All menu entries live under `HKCU:\Software\Classes\Directory\Background\shell\DevTools\shell\`.
Sub-items are keyed with numeric prefixes (`01terminal`, `02pwsh`, …) to control display order.

### Machine-specific files (git-ignored)
- `.env` — app paths, auto-generated by `setup-env.ps1`
- `.hidden-entries.json` — registry keys that `hide-duplicates.ps1` suppressed; read by `uninstall.ps1` to restore them

## Adding a New Tool

Four files must be edited in tandem:

1. **`setup-env.ps1`** — add a `Find-App` call for the new `APP_PATH` key
2. **`install.ps1`** — add an `Add-MenuItem` call (increment the numeric key prefix to set menu position)
3. **`dev-launcher.ps1`** — add a `switch` case for the new action name calling `Invoke-App`
4. **`hide-duplicates.ps1`** — add the app's own registry context menu paths to `$appToKeys` (only if the app installs its own context menu entry)

Then re-run `setup-env.ps1` and `install.ps1` to apply.

## Key Behaviors

- `setup-env.ps1` checks known install paths first, then falls back to `PATH` (skips `.cmd`/`.bat` wrappers — only accepts `.exe`/`.com`). Also searches Scoop (`~\scoop\apps\`).
- `install.ps1` skips any app whose `.env` value is `NOT_FOUND`.
- `dev-launcher.ps1` runs hidden (`-WindowStyle Hidden`) and shows a WScript popup on error.
- `hide-duplicates.ps1` suppresses duplicate entries in two ways: `LegacyDisable` for simple shell verb keys, and key deletion for COM shellex handlers (e.g. PowerToys Power Rename). Backup is saved to `.hidden-entries.json` and merges with any prior backup on each run.
- `uninstall.ps1` also cleans up legacy `CommandStore` entries from older installs.
